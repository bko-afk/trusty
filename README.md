# Страхование.Отзывы — тестовая версия

Каталог страховых компаний с рейтингами, отзывами и ответами на отзывы, статьями и минимальной админкой. Next.js (App Router) + TypeScript + Payload CMS 3 + PostgreSQL.

## Важно: ограничение этой сборки

Код написан и структурирован полностью, но **не был собран и запущен** в среде, где я его создавал — там нет доступа в интернет для `npm install` (заблокирован реестр npm и GitHub). Поэтому перед первым запуском нужно:

1. Установить зависимости (`npm install`) на своей машине или в CI/Vercel, где интернет есть.
2. Запустить и посмотреть, какие ошибки вылезут — на 90% это будут мелкие несостыковки версий Payload/Next (API этих пакетов быстро меняется), которые за 1-2 итерации легко поправить тем же агентом.

Дальше в этом README — как это сделать по шагам.

## Структура

- `src/payload.config.ts` — конфигурация Payload: коллекции, база данных, редактор.
- `src/collections/` — Companies (компании), InsuranceTypes (виды страхования), Reviews (отзывы), ReviewReplies (ответы на отзывы), Articles (статьи), Media (файлы), Users (админы).
- `src/app/(frontend)/` — публичные страницы: главная, каталог, карточка компании, отзывы, рейтинги, статьи, поиск, формы добавления компании/отзыва.
- `src/app/(payload)/` — служебные роуты админки и API (генерируются Payload автоматически, трогать не нужно).
- `src/seed/seed.ts` — скрипт с тестовыми данными (вымышленные компании и отзывы для проверки вёрстки и логики).
- `docker-compose.yml` + `Dockerfile` — контейнеры приложения и Postgres.

## Запуск локально

Нужен Node.js 20+ и Docker (для базы данных).

```bash
cp .env.example .env
# при желании поменяйте PAYLOAD_SECRET на случайную строку

docker compose up -d db      # поднимаем только Postgres
npm install
npm run generate:types       # генерирует src/payload-types.ts из коллекций
npm run seed                 # заполняет базу тестовыми данными
npm run dev
```

Откройте:
- `http://localhost:3000` — сайт
- `http://localhost:3000/admin` — админка (при первом заходе Payload предложит создать первого администратора)

## Что можно делать в админке

- Добавлять/редактировать страховые компании, переключать статус (черновик / опубликовано / скрыто), отмечать «подтверждённая компания».
- Управлять видами страхования (ОСАГО, КАСКО, ДМС и т.д.) — это категории каталога.
- Модерировать отзывы: на модерации / опубликован / скрыт / отклонён / спам.
- Добавлять ответы на отзывы от имени администрации или компании.
- Публиковать статьи и обзоры.
- Загружать логотипы и изображения (раздел Media), редактировать SEO-поля (title/description) у компаний и статей.

Заявки с публичных форм «Добавить компанию» и «Добавить отзыв» попадают в базу со статусом «черновик» / «на модерации» и не видны на сайте, пока их не опубликует администратор.

## Деплой бесплатной тестовой версии (Vercel + Neon)

1. Создайте репозиторий на GitHub и запушьте этот проект.
2. Зарегистрируйте бесплатную базу на [neon.com](https://neon.com) — скопируйте `DATABASE_URI`.
3. Импортируйте репозиторий в [vercel.com](https://vercel.com), укажите переменные окружения `DATABASE_URI`, `PAYLOAD_SECRET`, `NEXT_PUBLIC_SERVER_URL` (адрес вида `https://ваш-проект.vercel.app`).
4. После первого деплоя один раз выполните `npm run seed` локально, указав в `.env` тот же `DATABASE_URI`, что и на Neon — так тестовые данные попадут в облачную базу.
5. Для хранения загружаемых картинок в проде рекомендуется Cloudflare R2 (бесплатно 10 ГБ) через `@payloadcms/storage-s3` — переменные `S3_*` в `.env.example` уже подготовлены под это, адаптер нужно подключить в `payload.config.ts`, когда дойдёте до реальных загрузок.

## Перенос на Hostinger VPS

1. Возьмите VPS (от 8 ГБ RAM), при заказе выберите готовый образ **Dokploy** или **Coolify** — он есть в шаблонах Hostinger.
2. Через панель Dokploy/Coolify разверните этот репозиторий по `docker-compose.yml`.
3. Сначала можно оставить базу на Neon (просто укажите тот же `DATABASE_URI`) — перенесёте только приложение.
4. Когда будете готовы перенести и базу — поднимите контейнер `db` из `docker-compose.yml` на VPS и перелейте данные командой `pg_dump` (с Neon) → `pg_restore` (в контейнер на VPS), затем переключите `DATABASE_URI` на локальный адрес контейнера.

## Дальнейшая доработка

Это первая рабочая версия для проверки структуры и логики. Дальше по этой же кодовой базе можно:
- менять дизайн (Tailwind-классы в компонентах и `globals.css`);
- менять тексты и структуру полей в коллекциях;
- убирать/добавлять функциональность (например, лайки отзывов, жалобы, промокоды);
- добавлять новые виды страхования и поля через `src/collections/*.ts` — изменения схемы фиксируются миграциями Payload (`npm run migrate:create`).

Логотипы и изображения сейчас — SVG-заглушки в `public/placeholders/`. Замените их на свои через админку (раздел Media) или прямой заменой файлов.
